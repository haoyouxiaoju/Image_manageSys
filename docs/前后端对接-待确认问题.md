### 8. CLIP 图像编码分析（上传时调用）— P1

| # | 方法 | 路径 | 说明 | 前端当前占位 | 后端状态 |
|---|------|------|------|-------------|----------|
| 26 | `POST` | `/api/v1/clip/analyze` | 对单张图片做 CLIP 编码分析，返回语义特征 | `UploadView.vue` runCLIPAnalysis() 模拟 | 🔴 待开发 |

**当前前端流程（模拟版）**：
```
步骤 1：选择图片 → 预览
步骤 2：点击「CLIP AI 分析」→ 1.2s setTimeout 模拟编码
       → 返回模拟结果（视觉风格/主色调/内容密度/光线条件）
       → 自动填充表单（描述/标签/来源）
步骤 3：用户检查 CLIP 分析结果 + 修改表单 → 点击「确认上传」
```

**后端就绪后替换方案**：
```
步骤 2 改为：
  调 POST /api/v1/clip/analyze  { image: File }
  → Chinese-CLIP 图像编码器 → 512 维向量
  → 返回 { embedding_stored: true, features: { ... }, suggested_description: "...", suggested_tags: [...] }
  → 前端自动填充表单
```

**请求**：
- Content-Type: `multipart/form-data`
- 字段：`file`（图片文件）

**响应格式建议**：
```json
{
  "embedding_id": 123,
  "model": "chinese-clip-vit-b-16",
  "model_version": "1.0",
  // CLIP 可以返回的语义特征（通过向量与预定义标签计算相似度得出）：
  "features": {
    "style": "美食摄影",        // 与风格标签集计算最相似的标签
    "color_tone": "暖色调",      // 与色调标签集计算
    "scene": "室内",            // 与场景标签集计算
    "objects": ["咖啡", "杯子"] // 与物体标签集计算 Top-K
  },
  // 或者更简单的方式——直接返回一个自然语言描述（如果后端做了 Caption 模型）：
  "suggested_description": "一杯热咖啡特写，木质桌面背景，暖色调",
  "suggested_tags": ["咖啡", "热饮", "美食摄影", "暖色调"]
}
```

**对接要点**：
- 前端 `runCLIPAnalysis()` 中的 `setTimeout` 替换为 `assetsApi.clipAnalyze(formData)`
- 自动填充逻辑不变：`formDesc = response.suggested_description`
- CLIP 面板中的"风格/色调/密度/光线"将改为展示真实 CLIP 特征
- **注意**：CLIP 本身不直接输出"风格/色调"等分类标签，这些需要通过"用 CLIP 文本编码器编码预定义标签集 → 与图像向量计算余弦相似度 → 取 Top-K"的间接方式获得
- **简化方案**：如果暂时不做标签匹配，后端只需返回 `suggested_description` 和 `suggested_tags`，前端面板改为展示这些信息
