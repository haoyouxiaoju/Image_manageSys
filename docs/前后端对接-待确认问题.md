## 1. 素材管理

### Q1.1 素材详情是否增加编辑模式？

- **提出日期**：2026-05-07
- **状态**：✅ 已确认
- **问题**：当前素材详情页 `name` / `desc` / `source` 字段只读，仅标签可增删。是否需要加"编辑"按钮切换为可编辑状态？
- **前端影响**：加 `isEditing` 状态 → `el-input` 替换文本 → 调 `store.updateAsset()` 或 `PUT /api/v1/assets/{id}`
- **当前占位**：字段只读，标签可编辑

**决议**：需要增加编辑模式。素材详情页支持“查看/编辑”切换，`name`/`desc`/`source` 可编辑，提交走 `PUT /api/v1/assets/{id}`。

### Q1.2 素材上传的文件大小和格式限制

- **提出日期**：2026-05-07
- **状态**：✅ 已确认
- **问题**：前端当前限制 JPG/PNG/WebP，≤20MB。后端是否有额外限制？（如像素尺寸、DPI等）
- **前端影响**：上传校验逻辑

**决议**：MVP 统一限制为 `JPG/PNG/WebP` 且单文件 `<=20MB`。暂不增加 DPI/像素硬性限制，由后端做同口径校验并返回明确错误码。

- **提出日期**：2026-05-07
- **状态**：✅ 已确认
- **问题**：删除素材时，是直接物理删除（DB + 文件），还是标记 `is_deleted=true` 软删除？

## 3. 搜索

### Q3.1 全局搜索是否接入 CLIP 语义搜索？

- **提出日期**：2026-05-07
- **状态**：✅ 已确认
- **问题**：Header 全局搜索框目前用本地模糊匹配（名称/描述/CLIP描述包含关键词）。后续是否：
  - **先保持**：本地匹配（P0 阶段可用，无需后端）
  - **后升级**：跳搜索页 + 调 `POST /api/v1/search/text`（P1 接入 CLIP）
- **前端影响**：当前本地匹配已经可用；升级只需改 `onGlobalSearch` 逻辑
- **当前占位**：本地 `store.globalSearch` 筛选 `filteredAssets`

**决议**：按分阶段策略执行：P0 保持本地匹配，P1 再升级为跳转搜索页并调用 `POST /api/v1/search/text`（CLIP 语义检索）。

## 6. CLIP 集成

### Q6.1 上传后向量化是同步还是异步？

- **提出日期**：2026-05-07
- **状态**：✅ 已确认
- **问题**：上传素材后，Chinese-CLIP 编码是：
  - **同步**：上传接口等编码完成再返回（小文件快但大文件慢）
  - **异步**：上传接口立即返回，后台任务编码，前端轮询状态
- **前端影响**：同步模式简单；异步模式需要状态轮询 + "索引中"标记
- **当前占位**：模拟同步完成（假进度条）

**决议**：MVP 采用同步向量化（上传后同步编码）。当数据量或耗时达到阈值后再切异步任务模式。

---

- **提出日期**：2026-05-07
