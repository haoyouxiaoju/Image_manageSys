### 7. CLIP 图像编码与分析

| # | 方法 | 路径 | 说明 | 前端当前占位 | 后端状态 |
|---|------|------|------|-------------|----------|
| 18 | `GET` | `/api/v1/clip/status` | 模型状态（ready、last_error、model_name、model_version） | 无（UploadView 未调用） | 🟢 已就绪 |
| 19 | `POST` | `/api/v1/clip/analyze` | 对单张图片做 CLIP 编码，返回语义特征+建议标签 | `UploadView.vue` runCLIPAnalysis() setTimeout 模拟 | 🟢 已就绪 |

**实际后端响应格式**（`clip.py:17-24`）：
```json
{
  "model": "chinese-clip-vit-b-16",
  "model_version": "1.0",
  "embedding": [0.12, -0.34, ...],
  "embedding_dim": 512,
  "features": {
    "style": "美食摄影",
    "color_tone": "暖色调",
    "objects": ["咖啡", "杯子"]
  },
  "suggested_description": "一杯热咖啡特写，木质桌面背景，暖色调",
  "suggested_tags": ["咖啡", "热饮", "美食摄影", "暖色调"]
}
```

**当前前端流程（模拟版）**：
```
步骤 1：选择图片 → 预览
步骤 2：点击「CLIP AI 分析」→ 1.2s setTimeout 模拟编码
       → 返回模拟结果（视觉风格/主色调/内容密度/光线条件）
       → 自动填充表单（描述/标签/来源）
步骤 3：用户检查结果 + 修改表单 → 点击「确认上传」
```

**后端就绪后替换方案**：
```
步骤 2 改为：
  调 POST /api/v1/clip/analyze  { file: File }
  → Chinese-CLIP 图像编码器 → 512 维向量
  → 返回 { model, features, suggested_description, suggested_tags }
  → 前端自动填充表单
```

**额外说明**（commit db694d6 后）：
- `POST /api/v1/assets/upload` 在后端已集成 CLIP 同步分析——上传完成后即持久化 CLIP 结果到 `asset_clip_analysis` 表
- 前端可在上传前调用 `/clip/analyze` 做预分析（用于自动填表单），也可直接上传让后端自动处理
- `GET /api/v1/clip/status` 可用于检测模型是否真正加载（`ready=true` 表示 Chinese-CLIP 已启动）
