# 前端对接实施步骤 · 已完成

> 归档日期：2026-05-08 | 来源：`docs/前后端对接-待确认问题.md`「前端对接切换计划-已完成的步骤」

## 1. 认证模块切真实 API

`stores/auth.ts`：`USE_MOCK_AUTH = false`，登录走 `POST /api/v1/auth/login`，角色从 `/me` 响应读取。
- 登录响应含 `role` 字段（admin/editor/guest），前端据此控制 UI
- `/me` 返回 `role`，用于页面刷新后恢复角色状态

## 2. 素材 CRUD 走 API

`stores/assets.ts`：素材增删改查全部走 `assetsApi`，`BackendAsset → Asset` 映射。
- GET /api/v1/assets 分页+筛选+搜索
- GET /api/v1/assets/{id} 素材详情
- PUT /api/v1/assets/{id} 编辑元数据
- DELETE /api/v1/assets/{id} 删除素材
- 响应格式：`{ items: Asset[], total: number, page: number, page_size: number }`

## 3. 视图异步加载改造

`AssetsView.vue` / `AssetDetailView.vue`：接入 `LoadingSkeleton` + 异步加载，API 请求期间显示骨架屏。

## 4. 上传三步流程

`UploadView.vue`：三步流程（选择→CLIP分析→确认上传），FormData + axios 真实上传 `POST /api/v1/assets/upload`。

## 5. 注册模式

`LoginView.vue`：新增注册模式，对接 `POST /api/v1/auth/register`。

## 6. 语义搜索后端实现

`POST /api/v1/search/text` 全部后端工作就绪（plan `docs/plans/PLAN-20260508-search-text-p1.md` 验收项全部达标）：

- `backend/app/api/v1/endpoints/search.py` — 新增端点，路由 `/api/v1/search/text`
- `backend/app/services/clip_service.py` — 新增 `encode_text()` / `_encode_text_mock()` / `_encode_text_chinese_clip()` / `_ensure_ready()`
- `backend/app/repositories/clip_repository.py` — 新增 `list_ready_embeddings_assets()` 读取可检索向量数据
- 应用层余弦相似度排序 + 分页 `{ items: [{ asset, score }], total, page, page_size }`
- 测试覆盖：语义检索分页与排序
- 对接文档中 search/text 状态已改为 🟢 已就绪
